# Human Review Guide

## 原則

AI評価は作品と公開職務情報を整理する補助情報です。最終採用判断、自動不採用、候補者の自動非表示には使いません。候補者が20名未満の間は、検索精度ではなく機能動作だけを確認します。

## Curiosity 12軸

| 軸 | 確認する内容 |
|---|---|
| 構図 | 視線誘導、バランス、奥行き、意図ある焦点 |
| ライティング | 光源整合、自然光/人工光の階調、空気感 |
| マテリアル | 素材固有の反射、粗さ、質感の差 |
| 高級ブランド適性 | 過剰演出に頼らない静けさ、品位、精度 |
| インテリア理解 | 空間構成、家具、仕上げ、動線、スケール |
| ディテール | 納まり、接合、エッジ、小物、近景精度 |
| 仕上げ | 色、ノイズ、レタッチ、最終画の一貫性 |
| 技術適応力 | ツール、工程、修正、複数手法への適応可能性 |
| ホテル適性 | 滞在体験、時間帯、雰囲気、照明計画 |
| リテール適性 | 商品、ブランド、顧客動線、空間の主従 |
| 人工照明表現 | 器具、間接光、色温度、反射、複数光源 |
| デザイン理解 | 設計思想、素材選択、ブランド文脈の翻訳 |

各軸は説明、良い例、懸念例、重み、必須/参考を持ちます。初期versionの重み合計は100です。

## Rubric versionの公開

1. `評価基準`で現行12軸を確認する。
2. 変更する軸の説明・例・重み・必須区分を編集する。
3. 重み合計を100にする。
4. 変更理由を記録し、新versionを公開する。
5. 過去versionは変更しない。以降のAI評価だけが新version IDを保存する。

評価基準を変更した前後の結果を比較するときは、必ずrubric versionを揃えるか、version差を注記してください。

## Review Sampling

`Review Sampling`は次を自動抽出します。

- AI高得点候補
- AI低得点候補
- 直近のAI/人間差が15点以上
- データ品質70未満
- 複数Visual Searchで順位差が3以上

人間評価者は12軸を0〜100で入力し、観察根拠と未確認事項をコメントします。保存時にAI評価スナップショット、平均絶対差、抽出理由、評価者、日時、rubric versionを記録します。

## コメントの書き方

- 観察できる作品特徴と公開経歴だけを書く。
- 「ブランド経験あり」など未確認事項を断定せず、「公開情報では未確認」と書く。
- 国籍、年齢、性別、人種、障害、健康、家族状況など保護属性を推測しない。
- 「不採用」ではなく、仕事上確認すべきリスク、追加作品、面談質問を記録する。
- AIとの差が大きい場合は、画像不足、権利上送れなかった作品、rubric解釈、案件文脈のどれが原因かを確認する。

## 検索品質の読み方

- AI総合点: 候補者の作品評価。
- Scout適合点: 特定案件要件に対する相対評価。
- Visual Fit Score: 参考画像と作品傾向の類似性。
- AI/人間差: 同じrubric軸に対する平均絶対差。
- Precision@3 / @5: 人が想定した上位候補と実際の検索上位の一致率。

これらは別の指標です。単一指標だけで採用判断を行わず、候補者数、データ品質、画像数、rubric versionを併記してください。
