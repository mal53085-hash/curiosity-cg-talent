# dig Data Sources

## 方針

digは公開されていることと、収集・保存・再利用が許可されていることを区別する。新しい自動取得元は、利用規約、robots.txt、API契約、保存権、削除対応を確認してから有効化する。

| ソース | 現在の方式 | 自動取得 | Inbox | 注意事項 |
|---|---|---:|---:|---|
| Manual | URLと人手入力 | なし | 必須 | 登録者が公開情報と権利を確認 |
| LinkedIn | URL手動登録・CSV | 禁止 | 必須 | 大量自動取得・無断スクレイピング禁止 |
| Behance | 手動検索、契約済み検索API候補 | 既定無効 | 必須 | 保存権を含む契約を確認するまで自動保存しない |
| ArtStation | Phase 5で契約・規約確認後 | 未実装 | 必須 | API/robots.txt/利用規約確認 |
| CGArchitect | Phase 5で契約・規約確認後 | 未実装 | 必須 | 取得頻度と保存範囲を確認 |
| 個人サイト | URL手動登録、上限付き検索候補 | 制限付き | 必須 | robots.txt、利用規約、著作権を尊重 |
| Brave Search API | 検索結果の発見補助 | 設定時のみ | 必須 | 保存権を含む契約確認フラグが必要 |
| Visual Search参考画像 | 権利確認後の手動アップロード | なし | 対象外 | 非公開、30日保持、即時削除、システムは権利を保証しない |

## 共通制約

- Cronは1回最大5テーマ、1テーマ最大20件。
- 重複判定後も正式候補へ自動追加しない。
- LinkedInページ本文を自動取得しない。
- 取得失敗、規約未確認、キー未設定は`discovery_runs`へ秘密値なしで記録する。
- 削除依頼や公開停止に対応できるよう`source_url`と発見日時を保持する。
- Visual Search参考画像はDiscoveryソースへ転用せず、候補者発見の検索入力としてだけ使用する。
